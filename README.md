# 오늘 뭐해? 서면/전포 데이트 코스 추천 MVP

20대 초중반 대학생 커플을 위한 발표용 웹앱 MVP입니다. 엑셀 DB를 JSON으로 변환한 뒤, 사용자의 취향과 오늘 상황을 바탕으로 `식사 -> 카페/술집 -> 중간경유지 -> 지도 코스`를 추천합니다.

## 1. 설치 방법

```bash
npm install
```

## 2. `.env.local` 설정 방법

프로젝트 루트에 `.env.local` 파일을 만들고 Kakao Maps JavaScript 키를 넣습니다.

```env
NEXT_PUBLIC_KAKAO_JS_KEY=카카오지도_JS_API_KEY
```

API 키는 코드에 직접 넣지 않습니다. 키가 없거나 지도 로딩에 실패하면 앱은 장소 리스트를 보여주고 지도 영역에는 실패 메시지를 표시합니다.

## 3. 엑셀 DB 넣는 방법

아래 둘 중 하나를 `data/` 폴더에 넣습니다.

```text
data/final_db_with_coordinates.xlsx
data/final_db_with_keywords.xlsx
```

현재 프로젝트에는 `data/final_db_with_coordinates.xlsx`가 들어있습니다. 좌표가 없는 장소는 추천에는 사용할 수 있지만 지도 마커에서는 제외되고 `좌표 없음`으로 표시됩니다.

## 4. 엑셀을 JSON으로 변환

```bash
npm run convert
```

변환 결과는 아래 파일로 저장됩니다.

```text
data/places.json
```

## 5. 실행 방법

```bash
npm run dev
```

브라우저에서 보통 아래 주소로 접속합니다.

```text
http://localhost:3000
```

## 6. 추천 로직 설명

MVP에서는 외부 LLM API를 쓰지 않고 rule-based 추천을 사용합니다.

- 문장에서 `화해`, `시험`, `비`, `술`, `돈`, `조용히` 같은 태그를 추출합니다.
- 온보딩에서 저장한 분위기, 음식 선호, 회피 조건을 결합합니다.
- 장소역할이 현재 단계와 맞으면 가산점이 붙습니다.
- 추천상황키워드, 핵심키워드, 분위기/이용키워드가 사용자 조건과 맞으면 점수가 올라갑니다.
- 주의/제외키워드가 회피 조건과 겹치면 감점합니다.
- 운영시간이 휴무로 보이면 제외하거나 우선순위를 낮춥니다.
- 걸어서 이동하거나 많이 걷기를 싫어하면 거리 감점이 커집니다.

## 7. 지도 표시 방식

Kakao Maps JavaScript API를 사용합니다.

- 선택된 코스 장소를 번호 마커로 표시합니다.
- 마커 사이를 polyline으로 연결합니다.
- 정확한 도보 길찾기 API는 MVP에서 구현하지 않았습니다.
- 장소별 지도 보기 버튼은 DB의 카카오지도 링크 또는 네이버지도 링크로 이동합니다.

## 8. 현재 한계

- 정확한 도보 경로가 아니라 직선거리 기반 마커 연결선입니다.
- 운영시간 형식이 애매하면 `방문 전 확인 필요`로 처리합니다.
- AI 추천은 실제 LLM이 아니라 rule-based 태그 추천입니다.
- 엑셀 컬럼명이 크게 달라지면 변환 스크립트의 컬럼 매핑을 보완해야 합니다.

## 9. 추후 개선

- 실제 LLM API 연동
- 카카오 도보 길찾기 API 또는 외부 경로 API 연동
- 사용자 선택/거절 피드백 기반 추천 개선
- 날짜 선택과 실시간 영업 여부 검증
